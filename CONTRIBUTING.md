# Contributing

Thank you for your interest in contributing!

What we welcome
- Bug reports
- Bug fixes
- Documentation improvements
- New features and enhancements
- Ideas and design suggestions

How to contribute
1. Fork the repo and create a branch: `git checkout -b my-feature`
2. Make your changes, add tests (if applicable) and update documentation.
3. Commit with clear messages: `git commit -m "Short, descriptive message"`
4. Push your branch and open a Pull Request with a clear description of what you changed and why.

Reporting bugs
- Use the repository Issues tab to report bugs.
- Provide steps to reproduce, expected vs actual behavior, and any relevant logs or screenshots.

Code style and tests
- Follow the existing project style.
- Add tests where appropriate and ensure all checks pass before requesting review.

Maintainers
- Maintainers will review contributions and provide feedback; please be responsive to review comments.
